The `CCBundler` class handles dependency resolution and creates self-contained Lua bundles for ComputerCraft.

## Import

```typescript
import { CCBundler } from "@cc-ts/builder";
```

## Class: CCBundler

Bundles TypeScript modules into single Lua files with a custom module loader.

### Constructor

```typescript
new CCBundler(
  files: ProcessedFile[],
  program: ts.Program,
  emitHost: EmitHost,
  diagnostics: ts.Diagnostic[]
)
```

<ParamField path="files" type="ProcessedFile[]" required>
  Array of transpiled Lua files from TypeScript-to-Lua
  
  ```typescript
  interface ProcessedFile {
    fileName: string;
    code: string;
    sourceMapNode?: SourceNode;
    sourceFiles?: ts.SourceFile[];
  }
  ```
</ParamField>

<ParamField path="program" type="ts.Program" required>
  TypeScript program instance used for module resolution and configuration
</ParamField>

<ParamField path="emitHost" type="EmitHost" required>
  Emit host for file operations from TypeScript-to-Lua
</ParamField>

<ParamField path="diagnostics" type="ts.Diagnostic[]" required>
  Array to collect diagnostics during bundling
</ParamField>

#### Example

```typescript
import { CCBundler } from "@cc-ts/builder";
import * as ts from "typescript";

const bundler = new CCBundler(
  processedFiles,
  program,
  emitHost,
  diagnostics
);
```

### Methods

#### bundleModule()

Creates a complete bundle for a module and all its dependencies.

```typescript
bundleModule(moduleName: string): EmitFile | undefined
```

<ParamField path="moduleName" type="string" required>
  Module path to bundle (e.g., `"src.main"` or `"utils.helper"`)
</ParamField>

**Returns:** `EmitFile | undefined`

```typescript
interface EmitFile {
  outputPath: string;      // Output file path
  code: string;            // Bundled Lua code
  sourceMap: string;       // Source map JSON
  sourceFiles: ts.SourceFile[];  // Original TypeScript files
}
```

Returns `undefined` if the module is not found.

#### Example

```typescript
const bundle = bundler.bundleModule("src.main");

if (bundle) {
  console.log(`Output: ${bundle.outputPath}`);
  console.log(`Size: ${bundle.code.length} bytes`);
  
  // Write to file system
  fs.writeFileSync(bundle.outputPath, bundle.code);
  fs.writeFileSync(
    bundle.outputPath + ".map",
    bundle.sourceMap
  );
}
```

#### createModulePath()

Converts a file path to a Lua module path.

```typescript
createModulePath(pathToResolve: string): string
```

<ParamField path="pathToResolve" type="string" required>
  Absolute file path to convert
</ParamField>

**Returns:** Lua-style module path (e.g., `"src.utils.helper"`)

#### Example

```typescript
const modulePath = bundler.createModulePath("/project/src/utils/helper.ts");
// Returns: "src.utils.helper"
```

#### isBuiltInModule()

Checks if a module is considered built-in and should not be bundled.

```typescript
isBuiltInModule(moduleName: string): boolean
```

<ParamField path="moduleName" type="string" required>
  Module name to check
</ParamField>

**Returns:** `true` if the module is built-in

Built-in modules include:
- Modules starting with `cc.` (ComputerCraft APIs)
- `lualib_bundle` (TypeScript-to-Lua helpers)
- Modules in `compilerOptions.builtInModules` array

#### Example

```typescript
bundler.isBuiltInModule("cc.pretty");        // true
bundler.isBuiltInModule("lualib_bundle");    // true
bundler.isBuiltInModule("src.main");         // false
```

#### generateHeader()

Generates bundle metadata header.

```typescript
generateHeader(options: HeaderOptions): string
```

<ParamField path="options.entry" type="string" required>
  Entry module name
</ParamField>

<ParamField path="options.files" type="string[]" required>
  List of files included in the bundle
</ParamField>

<ParamField path="options.minify" type="boolean" required>
  Whether the bundle is minified
</ParamField>

**Returns:** Header comment string

#### Example

```typescript
const header = bundler.generateHeader({
  entry: "src.main",
  files: ["src/main.ts", "src/utils.ts"],
  minify: false,
});

console.log(header);
// -- Generated by @cc-ts/builder@1.0.0
// -- HeaderSchema: @cc-ts/builder - v0.0.1
// -- Entry: src.main
// -- Hash: {#Hash}
// -- BuildTime: 2024-01-15T10:30:00.000Z
// -- Files: src/main.ts, src/utils.ts
// -- Minified: false
```

## Bundle Structure

A bundled Lua file has the following structure:

```lua
-- Header (unless noHeader is true)
-- Generated by @cc-ts/builder@1.0.0
-- Entry: src.main
-- Files: src/main.ts, src/utils.ts

-- Optional: Extra paths setup
local ____defaultPath = package.path
package.path = ____defaultPath..";/rom/modules/?.lua"

-- Custom require() override
local ____modules = {}
local ____moduleCache = {}
local ____originalRequire = require
local function require(file, ...)
    if ____moduleCache[file] then
        return ____moduleCache[file].value
    end
    if ____modules[file] then
        local module = ____modules[file]
        ____moduleCache[file] = { value = module(file) }
        return ____moduleCache[file].value
    else
        if ____originalRequire then
            return ____originalRequire(file)
        else
            error("module '" .. file .. "' not found")
        end
    end
end

-- Module table
____modules = {
["src.main"] = function(...)
    -- Module code here
end,
["src.utils"] = function(...)
    -- Module code here
end,
}

-- Optional: Source map traceback
local __TS__SourceMapTraceBack = require("lualib_bundle").__TS__SourceMapTraceBack
__TS__SourceMapTraceBack(debug.getinfo(1).short_src, {...})

-- Entry point
return require("src.main", ...)
```

## Bundling Process

### Dependency Resolution

The bundler automatically resolves and includes all dependencies:

```typescript
// src/main.ts
import { helper } from "./utils/helper";
import { format } from "./utils/format";
import * as cc from "cc.pretty";  // Built-in, not bundled

// Bundler includes:
// - src.main
// - src.utils.helper
// - src.utils.format
// - lualib_bundle (if needed)
// Excludes: cc.pretty (built-in)
```

### Circular Dependencies

Circular dependencies are handled correctly:

```typescript
// a.ts
import { b } from "./b";
export const a = 1;

// b.ts
import { a } from "./a";
export const b = 2;

// Both modules are included without infinite loops
```

### Built-in Module Detection

Modules are excluded from bundling if they:

1. Start with `cc.` (ComputerCraft APIs)
2. Are named `lualib_bundle`
3. Are listed in `compilerOptions.builtInModules`

```json tsconfig.json
{
  "compilerOptions": {
    "outDir": "dist"
  },
  "cc-ts": {
    "builtInModules": ["mylib", "external.api"]
  }
}
```

## Advanced Features

### Source Maps

Source maps are generated for debugging:

```typescript
const bundle = bundler.bundleModule("src.main");

// Write source map
fs.writeFileSync(
  bundle.outputPath + ".map",
  bundle.sourceMap
);
```

### Source Map Traceback

When `sourceMapTraceback` is enabled, runtime errors show TypeScript locations:

```json tsconfig.json
{
  "compilerOptions": {
    "sourceMapTraceback": true
  }
}
```

Error output:
```
main.lua:15: in function 'helper'
  src/utils.ts:23
```

### Minification

Bundles can be minified using luamin:

```typescript
const options: CompilerOptions = {
  minify: true,
};

// Output is minified:
// local a={}local b={}local c=require...
```

### Extra Paths

Add custom module search paths:

```json tsconfig.json
{
  "cc-ts": {
    "extraPaths": ["/rom/modules", "/disk/libs"]
  }
}
```

Generated code:
```lua
local ____defaultPath = package.path
package.path = ____defaultPath..";/rom/modules/?.lua;/rom/modules/?/init.lua;/disk/libs/?.lua;/disk/libs/?/init.lua"
```

## Complete Example

```typescript
import { CCBundler } from "@cc-ts/builder";
import * as tstl from "@cc-ts/typescript-to-lua";
import * as ts from "typescript";
import * as fs from "fs";

// 1. Create TypeScript program
const program = ts.createProgram({
  rootNames: ["src/main.ts"],
  options: {
    target: ts.ScriptTarget.ESNext,
    module: ts.ModuleKind.CommonJS,
    outDir: "dist",
  },
});

// 2. Transpile to Lua
const transpiler = new tstl.Transpiler();
const { diagnostics: emitDiagnostics } = transpiler.emit({ program });

// 3. Get processed files
const sourceFiles = program.getSourceFiles()
  .filter(sf => !sf.isDeclarationFile);

const processedFiles: tstl.ProcessedFile[] = [];
for (const sourceFile of sourceFiles) {
  const emitResult = transpiler.emitFile(sourceFile);
  processedFiles.push({
    fileName: sourceFile.fileName,
    code: emitResult.code,
    sourceMapNode: emitResult.sourceMapNode,
    sourceFiles: [sourceFile],
  });
}

// 4. Create bundler
const diagnostics: ts.Diagnostic[] = [];
const bundler = new CCBundler(
  processedFiles,
  program,
  transpiler.emitHost,
  diagnostics
);

// 5. Bundle main module
const modulePath = bundler.createModulePath(
  "/absolute/path/to/src/main.ts"
);

const bundle = bundler.bundleModule(modulePath);

if (bundle) {
  // 6. Write output
  fs.writeFileSync(bundle.outputPath, bundle.code);
  fs.writeFileSync(
    bundle.outputPath + ".map",
    bundle.sourceMap
  );
  
  console.log(`✓ Bundle created: ${bundle.outputPath}`);
  console.log(`  Size: ${bundle.code.length} bytes`);
  console.log(`  Files: ${bundle.sourceFiles.length}`);
} else {
  console.error(`✗ Failed to bundle module: ${modulePath}`);
}

// 7. Check diagnostics
if (diagnostics.length > 0) {
  diagnostics.forEach(d => {
    console.error(ts.formatDiagnostic(d, {
      getCurrentDirectory: () => process.cwd(),
      getCanonicalFileName: f => f,
      getNewLine: () => "\n",
    }));
  });
}
```

## Troubleshooting

### Module Not Found

If `bundleModule()` returns `undefined`:

1. Check the module path format (use dots, not slashes)
2. Ensure the file was included in the transpilation
3. Verify the module is not filtered by `ignoreAsEntryPoint`

```typescript
// Incorrect
bundler.bundleModule("src/main");  // ✗

// Correct
bundler.bundleModule("src.main");  // ✓
```

### Missing Dependencies

If dependencies are missing from the bundle:

1. Ensure imports use correct paths
2. Check that modules are not marked as built-in
3. Verify TypeScript's module resolution is working

### Large Bundle Size

To reduce bundle size:

1. Enable minification: `minify: true`
2. Mark external modules as built-in
3. Use `ignoreAsEntryPoint` to prevent duplicate bundles
4. Remove unused imports
