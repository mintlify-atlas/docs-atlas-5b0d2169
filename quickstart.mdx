Get up and running with CC-TS by building a simple ComputerCraft turtle program. This guide walks you through creating a project, writing TypeScript code, and running it in-game.

## Create your project

Use `create-cc-ts` to scaffold a new project with all the configuration you need:

<CodeGroup>

```bash bun
bun create cc-ts my-turtle-miner
cd my-turtle-miner
```

```bash npm
npm create cc-ts@latest my-turtle-miner
cd my-turtle-miner
```

```bash yarn
yarn create cc-ts my-turtle-miner
cd my-turtle-miner
```

```bash pnpm
pnpm create cc-ts my-turtle-miner
cd my-turtle-miner
```

</CodeGroup>

<Note>
Choose **Bundled App** when prompted. This gives you a complete development setup with watch mode and hot reload.
</Note>

## Project structure

Your new project includes everything you need:

```
my-turtle-miner/
├── src/
│   └── index.ts          # Your main entry point
├── dist/                 # Compiled Lua output (generated)
├── tsconfig.json         # TypeScript configuration
├── package.json          # Dependencies and scripts
└── .gitignore
```

## Write your first program

Open `src/index.ts` and replace the contents with this simple turtle mining program:

```typescript
import { Event } from '@cc-ts/helpers/event'

interface Position {
  x: number
  y: number
  z: number
}

class TurtleMiner {
  private position: Position = { x: 0, y: 0, z: 0 }
  private fuel: number = turtle.getFuelLevel()

  async mine(length: number): Promise<void> {
    print(`Starting mining operation for ${length} blocks`)
    
    for (let i = 0; i < length; i++) {
      // Check fuel
      if (this.fuel < 1) {
        print('Out of fuel!')
        return
      }

      // Dig forward and move
      while (turtle.detect()) {
        turtle.dig()
        await Event.pullFiltered('turtle_inventory')
      }

      if (turtle.forward()) {
        this.position.x += 1
        this.fuel -= 1
        print(`Position: ${this.position.x}, Fuel: ${this.fuel}`)
      } else {
        print('Blocked! Cannot move forward.')
        break
      }
    }

    print('Mining complete!')
  }

  async returnHome(): Promise<void> {
    print('Returning to starting position...')
    turtle.turnLeft()
    turtle.turnLeft()
    
    for (let i = 0; i < this.position.x; i++) {
      turtle.forward()
    }
    
    print('Back at home!')
  }
}

// Main execution
async function main() {
  const miner = new TurtleMiner()
  await miner.mine(10)
  await miner.returnHome()
}

main()
```

This program demonstrates several TypeScript features working in ComputerCraft:
- **Classes** with private fields and methods
- **Interfaces** for type safety
- **Async/await** for event handling
- **Type annotations** that catch errors at compile time

## Configuration

Your `tsconfig.json` is pre-configured for ComputerCraft:

```json
{
  "compilerOptions": {
    "target": "esnext",
    "lib": ["esnext"],
    "moduleResolution": "node",
    "types": ["@jackmacwindows/craftos-types"],
    "strict": true
  },
  "tstl": {
    "luaTarget": "5.2",
    "luaLibImport": "require-minimal"
  }
}
```

Key settings:
- **luaTarget**: `5.2` - ComputerCraft uses Cobalt (Lua 5.2)
- **types**: Includes ComputerCraft API types for autocomplete
- **strict**: Enables full TypeScript type checking

## Start the dev server

Run the development server to compile your code and serve it over HTTP:

<CodeGroup>

```bash bun
bun run dev
```

```bash npm
npm run dev
```

```bash yarn
yarn dev
```

```bash pnpm
pnpm dev
```

</CodeGroup>

You'll see output like:

```
Starting HTTP server on port 8080
Watching for file changes...
Compilation complete: src/index.ts -> dist/index.lua
```

The builder will:
1. Compile TypeScript to Lua
2. Bundle all dependencies
3. Watch for changes and recompile automatically
4. Serve files at `http://localhost:8080/dist/`

## Load in ComputerCraft

In your ComputerCraft computer or turtle, download and run your program:

```lua
-- Download the compiled Lua
http.request("http://YOUR_IP:8080/dist/index.lua")
local event, url, response = os.pullEvent("http_success")
local code = response.readAll()
response.close()

-- Save to a file
local file = fs.open("miner.lua", "w")
file.write(code)
file.close()

-- Run it
shell.run("miner")
```

Replace `YOUR_IP` with your computer's local IP address (find it with `ipconfig` on Windows or `ifconfig` on Mac/Linux).

<Tip>
For easier testing, you can copy the compiled Lua directly from `dist/index.lua` into your ComputerCraft computer directory.
</Tip>

| `clean` | Remove `dist/` directory |

## Troubleshooting

<AccordionGroup>
  <Accordion title="Compilation errors about missing types">
    Make sure you have the ComputerCraft types installed:
    
    ```bash
    npm install --save-dev @jackmacwindows/craftos-types
    ```
    
    Check that your `tsconfig.json` includes them in the `types` array.
  </Accordion>
  
  <Accordion title="Cannot connect to dev server from ComputerCraft">
    - Verify your computer's IP address is correct
    - Check that port 8080 is not blocked by a firewall
    - Make sure the dev server is running (`bun run dev`)
    - Try using `127.0.0.1` if testing on the same machine
  </Accordion>
  
  <Accordion title="Turtle program doesn't work in-game">
    - Check the Lua output in `dist/index.lua` for errors
    - Ensure the turtle has fuel (`turtle.refuel()`)
    - Verify the turtle has an inventory to collect blocks
    - Look at the in-game console for error messages
  </Accordion>
  
  <Accordion title="Build is slow or uses too much memory">
    - Use `incremental` mode for faster rebuilds (enabled by default in watch mode)
    - Exclude unnecessary files with `ignoreAsEntryPoint` in your config
    - Consider splitting large projects into multiple entry points
  </Accordion>
</AccordionGroup>
